## 已修复问题明细

### P0 严重 Bug（5/5）

#### 1. web_app.py `/api/ocr` 调用 `_auto_ocr` 参数不匹配 ✅
**文件**: `web/web_app.py`
**修复**: 移除了 `_auto_ocr(full_path, ocr_txt_path)` 的第二个参数，改为 `_auto_ocr(full_path)`。OCR 输出路径通过 `os.path.splitext()` 推导。

#### 2. copaw_skill.py `rag_search_by_keyword()` 引用不存在的 `vectorstore` ✅
**文件**: `core/copaw_skill.py`
**修复**: 将 `kb.vectorstore.similarity_search(keyword, k=top_k)` 改为 `kb.search(keyword, top_k=top_k)`。

#### 3. copaw_skill.py `rag_status()` / `rag_list_documents()` 访问错误字典键 ✅
**文件**: `core/copaw_skill.py`
**修复**: `KnowledgeBase.get_status()` 返回的 `documents` 是路径字符串列表，已修改循环变量为 `doc_path`，移除了不存在的 `doc['path']` 和 `doc['chunks']` 访问。

#### 4. query_with_sources_fixed.py 废弃文件 ✅
**文件**: `core/query_with_sources_fixed.py`
**修复**: 已删除该文件。该文件依赖 ollama 和 `self.vectorstore`，与当前 ChromaDB + LM Studio 架构完全不兼容。

#### 5. core_qdrant.py `_generate_point_id` 整数溢出风险 ✅
**文件**: `core/core_qdrant.py`
**修复**: 将整数 ID 改为字符串 UUID（MD5 哈希），彻底消除 chunk_idx >= 1000 时的 ID 冲突问题。

---

### P1 高优先级问题（6/6）

#### 6. Embedding 错误时返回全零向量 ✅
**文件**: `core/core.py`
**修复**: `LMStudioEmbeddings._embed()` 在 Embedding 失败时改为抛出异常，而不是返回 `[0.0] * 2560`。避免全零向量污染 ChromaDB 检索结果。

#### 7. PDF OCR 路径大小写替换 bug ✅
**文件**: `core/core.py`
**修复**: 新增 `_get_ocr_txt_path(filepath)` 辅助方法，使用 `os.path.splitext()` 生成 OCR 文本路径，不依赖字符串替换。同时修复了 `_auto_ocr()` 和 `_scan_documents()` 中的相关逻辑。

#### 8. `_split_text` 分块时可能产生无限循环 ✅
**文件**: `core/core.py`
**修复**: 在 `_split_text()` 中添加了 `chunk_overlap >= chunk_size` 的自动修正，并确保 `start` 每次至少前进 1 个字符。

#### 9. Web API 路径遍历攻击 ✅
**文件**: `web/web_app.py`
**修复**: 新增 `secure_document_path()` 函数，在 `api_delete` 和 `api_ocr_file` 中验证文件路径是否在 `DOCUMENTS_PATH` 内部，防止 `../` 路径遍历攻击。

#### 10. `/api/upload` 无文件大小限制 ✅
**文件**: `web/web_app.py`
**修复**: 设置 `app.config['MAX_CONTENT_LENGTH'] = 500 * 1024 * 1024`，限制上传文件大小为 500MB。

#### 11. Excel 数值类型转换 bug ✅
**文件**: `core/excel_analyzer.py`
**修复**: 在过滤条件中使用 `pd.to_numeric(str(value).replace(',', ''), errors='coerce')` 统一处理数值转换，支持千分位、小数和科学计数法。
