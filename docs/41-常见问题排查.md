## 安装问题

### 问题 1：需要管理员权限

**症状：**
```
⚠️  需要管理员权限运行
请右键点击 install.bat，选择"以管理员身份运行"
```

**解决：**
1. 关闭当前窗口
2. 右键点击 `install.bat`
3. 选择"以管理员身份运行"

---

### 问题 2：VC++ 运行库缺失

**症状：**
```
❌ Visual C++ 运行库未检测到
ImportError: DLL load failed while importing core
```

**解决：**
```batch
# 运行 VC++ 修复工具
D:\RAG_Knowledge_Base\fix_vcredist.bat

# 或手动安装
# 下载：https://aka.ms/vs/17/release/vc_redist.x64.exe
# 安装后重启系统
```

---

### 问题 3：依赖包安装失败

**症状：**
```
❌ 依赖包验证失败
ERROR: Could not find a version that satisfies the requirement
```

**解决：**
```batch
# 方式 1：使用离线包
D:\RAG_Knowledge_Base\download_packages.bat

# 方式 2：在线安装
pip install -r D:\RAG_Knowledge_Base\requirements.txt

# 方式 3：逐个安装
pip install chromadb flask pymupdf rapidocr
```

## 启动问题

### 问题 1：Web 服务无法启动

**症状：**
```
❌ Web 服务启动失败
端口 5000 被占用
```

**解决：**
```batch
# 方式 1：查看占用端口的进程
netstat -ano | findstr ":5000"
taskkill /F /PID [进程 PID]

# 方式 2：修改端口
# 编辑 config.yaml
web_port: 5001

# 方式 3：停止所有服务后重启
D:\RAG_Knowledge_Base\stop_all.bat
D:\RAG_Knowledge_Base\start_all.bat
```

---

### 问题 2：LM Studio 未运行

**症状：**
```
❌ LM Studio 未运行 (端口 1234)
请先启动 LM Studio 并加载模型
```

**解决：**
1. 启动 LM Studio 应用
2. 加载模型 `qwen/qwen3.5-9b`
3. 启动本地服务器（端口 1234）
4. 确认绿色状态指示器

---

### 问题 2.1：API Key 认证失败

**症状：**
```
❌ LM Studio API 认证失败 (401 Unauthorized)
Invalid API Key
```

**原因：** LM Studio 开启了 API Key 认证，但配置文件中的 API_KEY 不正确

**解决：**
1. 打开 LM Studio，查看 API Key 设置
2. 编辑 `config\.env` 文件
3. 将 `API_KEY` 设置为正确的值：
   ```bash
   # config/.env 文件
   API_KEY=your-actual-api-key-here
   ```
4. 重启 Web 服务或 RAG 工具

---

### 问题 3：Flask 依赖缺失

```
ModuleNotFoundError: No module named 'flask'
```

**解决：**
```batch
# 安装 Flask
pip install flask flask-cors

# 或使用启动脚本自动安装
D:\RAG_Knowledge_Base\web\启动 Web 界面.bat
```

## 查询问题

### 问题 1：查询无结果

**症状：**
```
抱歉，知识库中没有找到相关信息
```

**原因：**
1. 知识库为空
2. 问题与文档不相关
3. 相似度阈值太高

**解决：**
```python
# 检查知识库状态
python D:\RAG_Knowledge_Base\core\core.py status

# 如果文档数为 0，刷新索引
python D:\RAG_Knowledge_Base\core\core.py refresh

# 降低相似度阈值
# 编辑 config.yaml
similarity_threshold: 0.2
```

---

### 问题 2：答案不准确

**症状：**
- 答案与问题不相关
- 检索结果质量差

**解决：**
```yaml
# 调整参数
top_k: 3              # 增加检索数量
similarity_threshold: 0.2  # 降低阈值
```

**优化问题：**
```
❌ 模糊：电费
✅ 具体：2026 年 2 月的电费金额是多少
```

---

### 问题 3：上下文超长错误

**症状：**
```
Error: n_keep: 22036 >= n_ctx: 16384
```

**原因：** 检索文档太多，超过模型上下文窗口

**解决：**
```yaml
# 方式 1：减少 top_k（推荐）
top_k: 2

# 方式 2：增大模型上下文窗口
# 在 LM Studio 中设置 Context Length: 32768
```

---

### 问题 4：查询超时

**症状：**
```
Request timeout
查询时间超过 60 秒
```

**解决：**
```yaml
# 减少检索数量
top_k: 2

# 关闭自动刷新
auto_refresh: false

# 检查 LM Studio 响应速度
# 确保模型已加载完成
```

## 索引问题

### 问题 1：索引创建失败

**症状：**
```
❌ 索引创建失败
Collection 不存在
```

**解决：**
```batch
# 强制重建索引
python D:\RAG_Knowledge_Base\core\core.py refresh --force

# 检查 ChromaDB 路径
dir "D:\RAG_Knowledge_Base\documents\.rag\chroma_db"
```

---

### 问题 2：文档未索引

**症状：**
- 添加了新文档但查询不到
- 文档数量不更新

**解决：**
```batch
# 手动刷新索引
python D:\RAG_Knowledge_Base\core\core.py refresh

# Web 界面点击"刷新索引"按钮

# 检查文档格式
# 支持：PDF, DOCX, TXT, Excel, PPTX
```

---

### 问题 3：索引重复

**症状：**
- 文档数量异常多
- 相同内容多次出现

**解决：**
```batch
# 删除向量数据库
rmdir /s /q "D:\RAG_Knowledge_Base\documents\.rag\chroma_db"

# 重新创建索引
python D:\RAG_Knowledge_Base\core\core.py refresh --force
```

## OCR 问题

### 问题 1：OCR 识别失败

**症状：**
```
❌ OCR 识别失败
ImportError: No module named 'paddle'
```

**解决：**
```batch
# 安装 RapidOCR
pip install rapidocr onnxruntime

# 安装 VC++ 运行库
D:\RAG_Knowledge_Base\fix_vcredist.bat

# 验证安装
python -c "from rapidocr import RapidOCR; print('rapidocr: OK')"
```

---

### 问题 2：OCR 准确率低

**症状：**
- 识别结果错别字多
- 文字顺序混乱

**解决：**
1. 提高原图质量（重新扫描，300 DPI）
2. 确保文字方向水平
3. 避免模糊和倾斜
4. 手动校对重要内容

---

### 问题 3：OCR 速度慢

**症状：**
- 单页超过 10 秒
- CPU 占用 100%

**解决：**
1. 降低图片分辨率（建议 300 DPI）
2. 关闭其他程序
3. 首次运行需下载模型（等待完成）

---

## 性能问题

### 问题 1：系统响应慢

**症状：**
- 查询时间超过 10 秒
- Web 界面卡顿

**解决：**
```yaml
# 优化配置
top_k: 2              # 减少检索数量
auto_refresh: false   # 关闭自动刷新

# 硬件优化
# - 使用 SSD
# - 增加内存到 16GB
# - 关闭其他程序
```

---

### 问题 2：内存不足

**症状：**
```
MemoryError
系统提示内存不足
```

**解决：**
```batch
# 增加虚拟内存（页面文件）
# 控制面板 → 系统 → 高级系统设置
# 性能 → 设置 → 高级 → 虚拟内存
# 设置为物理内存的 1.5-2 倍

# 关闭其他程序
# 重启系统
```

---

### 问题 3：磁盘空间不足

**症状：**
```
No space left on device
```

**解决：**
```batch
# 清理临时文件
del /q %TEMP%\*

# 清理旧日志
del /q D:\RAG_Knowledge_Base\logs\*.log

# 移动文档到其他盘
# 或增加磁盘空间
```

## 📋 故障排查流程

### 标准流程

```
1. 运行环境检测
   ↓
2. 检查服务状态
   ↓
3. 查看错误日志
   ↓
4. 根据错误信息定位问题
   ↓
5. 应用解决方案
   ↓
6. 验证问题是否解决
```

### 快速诊断脚本

```batch
@echo off
echo ╔════════════════════════════════════════════════════╗
echo ║       RAG 知识库 - 快速诊断                        ║
echo ╚════════════════════════════════════════════════════╝
echo.

REM 检查服务
netstat -ano | findstr ":1234" >nul 2>&1 && echo ✅ LM Studio: 运行中 || echo ❌ LM Studio: 未运行
netstat -ano | findstr ":5000" >nul 2>&1 && echo ✅ Web 服务：运行中 || echo ❌ Web 服务：未运行

REM 检查索引
python D:\RAG_Knowledge_Base\core\core.py status 2>nul | findstr "文档数"

REM 检查依赖
python -c "import chromadb, flask, pymupdf" 2>nul && echo ✅ 依赖包：正常 || echo ❌ 依赖包：缺失

echo.
echo 如发现问题，请查看对应文档解决
pause
```
