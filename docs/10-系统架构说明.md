## 🎯 系统概述

RAG（Retrieval-Augmented Generation）知识库系统是一个基于本地文档的智能问答系统，通过检索增强生成技术，实现基于私有知识的智能问答。

### 核心能力

- ✅ **文档索引** - 支持 PDF/DOCX/TXT/Excel/PPTX 多格式
- ✅ **语义检索** - 基于向量相似度的智能检索
- ✅ **智能问答** - 结合检索结果的 AI 答案生成
- ✅ **OCR 识别** - 图片 PDF 文字识别
- ✅ **多端访问** - Web/命令行/API/助手集成

## 🔄 核心工作流程

### 1. 文档索引流程

```
文档上传 → 格式检测 → 文本提取 → 文本分块 → 向量化 → 存储到 ChromaDB
   │
   ├─ PDF → PyMuPDF → 文本
   ├─ DOCX → python-docx → 文本
   ├─ Excel → openpyxl → 文本
   ├─ PPTX → python-pptx → 文本
   └─ 图片 PDF → OCR → 文本
```

### 2. 查询问答流程

```
用户问题 → 向量化 → 相似度检索 → 获取 top_k 文档块 → 构建提示词 → LLM 生成答案 → 返回结果
                                      │
                                      └─ 标注来源文档
```

### 3. OCR 处理流程

```
图片 PDF → 检测是否图片 → PDF 转图片 → RapidOCR 识别 → 生成 _ocr.txt → 索引
```

## 🔧 核心组件说明

### 1. RAG Core Engine (core.py)

**功能：** 核心引擎，提供所有 RAG 功能

**主要类：**
```python
class RAGKnowledgeBase:
    - __init__()          # 初始化
    - refresh_index()     # 刷新索引
    - query()             # 查询
    - get_status()        # 状态
    - ocr_pdf()           # OCR 处理
```

**关键方法：**
```python
# 查询示例
result = rag.query(
    "问题内容",
    top_k=2,           # 检索文档数
    auto_refresh=False # 不自动刷新
)
```

---

### 2. Web API (web_app.py)

**功能：** 提供 RESTful API 和 Web 界面

**API 端点：**
| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/query` | POST | 查询知识库 |
| `/api/refresh` | POST | 刷新索引 |
| `/api/status` | GET | 获取状态 |
| `/api/files` | GET | 文件列表 |
| `/api/files/delete` | POST | 删除文件 |
| `/api/files/ocr` | POST | OCR 处理 |

**请求示例：**
```javascript
// 查询
POST /api/query
{
  "query": "问题内容",
  "top_k": 2,
  "auto_refresh": false
}

// 响应
{
  "answer": "答案内容",
  "sources": [
    {"file": "文档.pdf", "content": "..."}
  ],
  "stats": {
    "prompt_tokens": 1000,
    "completion_tokens": 200
  }
}
```

---

### 3. ChromaDB 向量数据库

**功能：** 存储文档向量和元数据

**数据结构：**
```python
{
  "id": "unique_id",
  "embedding": [0.1, 0.2, ...],  # 2560 维向量
  "metadata": {
    "file": "文档.pdf",
    "chunk_id": 1,
    "text": "文本内容"
  }
}
```

**存储位置：**
```
D:\RAG_Knowledge_Base\documents\.rag\chroma_db\
```

---

### 4. LM Studio AI 模型

**功能：** 提供 LLM 和 Embedding 服务

**配置：**
```yaml
llm:
  base_url: http://127.0.0.1:1234
  model: qwen/qwen3.5-9b
  context_length: 16384
  # API Key 认证（如果 LM Studio 开启了认证）
  api_key: your-api-key-here

embedding:
  base_url: http://127.0.0.1:1234
  model: text-embedding-qwen3-embedding-4b
  dimension: 2560
  # API Key 认证（如果 LM Studio 开启了认证）
  api_key: your-api-key-here
```

**API Key 配置说明：**
- 如果 LM Studio 开启了 API Key 认证，需要在 `.env` 文件中配置 `API_KEY`
- 系统会自动在所有 LM Studio API 请求中添加 `Authorization: Bearer xxx` header
- 配置文件位置：`config\.env`

## 📈 性能指标

### 典型性能

| 操作 | 耗时 | 说明 |
|------|------|------|
| 查询（冷启动） | 5-10 秒 | 首次查询 |
| 查询（热启动） | 2-5 秒 | 后续查询 |
| 索引刷新（10 文档） | 30-60 秒 | 增量更新 |
| OCR 处理（1 页） | 2-5 秒 | 图片识别 |

### 优化建议

1. **减少 top_k** - 默认 2，避免上下文超长
2. **关闭 auto_refresh** - 只在添加文档后手动刷新
3. **使用 SSD** - 提升向量检索速度
4. **增加内存** - 建议 16GB+
