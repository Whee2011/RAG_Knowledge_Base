## 已修复问题明细

### N1. [P1] `embed_documents()` 单条失败导致整批失败 ✅

**文件**: `core/core.py`

**问题**: `LMStudioEmbeddings.embed_documents()` 对文本列表逐条调用 `_embed()`，单条失败即中断整个批处理，导致文档索引完全失败。

**修复**: 在 `embed_documents()` 中增加重试机制（默认重试 2 次）。单条文本失败时会自动重试，仍然失败才抛出异常并指明具体是哪条文本失败。

```python
def embed_documents(self, texts: List[str], max_retries: int = 2) -> List[List[float]]:
    embeddings = []
    for i, text in enumerate(texts):
        last_error = None
        for attempt in range(max_retries + 1):
            try:
                result = self._embed(text)
                embeddings.append(result)
                break
            except Exception as e:
                last_error = e
                if attempt < max_retries:
                    print(f"[Warning] ... 正在重试...")
                else:
                    raise Exception(f"第 {i+1}/{len(texts)} 条文本 Embedding 失败...") from last_error
    return embeddings
```

**同步修复**: `core/core_qdrant.py` 和 `core/core_chromadb.py` 中的 `LMStudioEmbeddings` 也同步增加了相同逻辑。

### N3. [P2] `secure_document_path()` 在 Windows 上大小写敏感 ✅

**文件**: `web/web_app.py`

**问题**: 路径合法性检查使用 `startswith()` 做区分大小写的比较，在 Windows 上可能通过大小写变换绕过路径遍历防护。

**修复**: 将路径比较改为不区分大小写：

```python
normalized_doc_path = os.path.normpath(DOCUMENTS_PATH).lower()
if not full_path.lower().startswith(normalized_doc_path):
    raise ValueError('非法路径')
```

### N4. [P2] `core_qdrant.py` `_chunk_text` 缺少无限循环保护 ✅

**文件**: `core/core_qdrant.py`

**问题**: `core.py` 的 `_split_text()` 已添加 `chunk_overlap >= chunk_size` 保护和 `start + 1` 最小前进保护，但 `core_qdrant.py` 的 `_chunk_text()` 仍使用 `start = end - self.chunk_overlap`，在错误配置下会产生无限循环。

**修复**: 在 `_chunk_text()` 中同步添加：
- `chunk_overlap >= chunk_size` 自动修正
- `end = min(start + self.chunk_size, text_length)` 防止越界
- `start = max(next_start, start + 1)` 确保每次至少前进 1 字符

### 原 #15. [P2] `hybrid_search.py` 向量距离未归一化 ✅

**文件**: `core/hybrid_search.py`

**问题**: `_vector_search()` 返回的余弦距离（越小越好）在 `_fuse_results()` 中被直接使用，而 `_normalize_scores(reverse=True)` 从未被调用，导致向量距离方向与 RRF 分数方向不一致，降低混合检索质量。

**修复**: 在 `search()` 方法中，对向量检索结果调用 `_normalize_scores(reverse=True)` 将距离转换为正向分数，对 BM25 结果调用 `_normalize_scores(reverse=False)` 归一化到 0-1 区间。

```python
# 将向量距离（越小越好）归一化为正向分数（越大越好）
if vector_results.get('scores'):
    vector_results['scores'] = self._normalize_scores(vector_results['scores'], reverse=True)

# 将 BM25 分数（越大越好）归一化为 0-1 区间
if bm25_results.get('scores'):
    bm25_results['scores'] = self._normalize_scores(bm25_results['scores'], reverse=False)
```
