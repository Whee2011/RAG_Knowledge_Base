## 二、本轮新发现的问题

### N6. [P2] core_qdrant.py _load_pdf() 仍使用不安全的字符串替换生成 OCR 路径
**文件**: core/core_qdrant.py 第 218 行
`python
ocr_txt_path = filepath.replace('.pdf', '_ocr.txt')
`
**问题**: 整改报告v2声称 N5 修复已同步到 core_chromadb.py，但遗漏了 core_qdrant.py。该文件的 _load_pdf() 仍然使用 ilepath.replace('.pdf', '_ocr.txt') —— 这正是原审计 P1 #7 中指出的大小写替换 bug。当文件名为 eport.PDF 时，.replace('.pdf', ...) 不会匹配。
**影响**: Qdrant 版本加载 OCR 文本文件时可能失败（文件路径不匹配）。
**建议**: 在 _load_pdf() 中使用 ase, _ = os.path.splitext(filepath); return base + '_ocr.txt' 替换。

### N7. [P2] add_documents() 未捕获 embed_documents() 异常，单条文本失败将中断整批索引
**文件**: core/core.py 第 684 行
`python
embeddings = self.embeddings.embed_documents(all_chunks)
`
**问题**: _embed() 现在在失败时抛出异常而非返回全零向量。但 dd_documents() 调用 embed_documents() 时没有 try/except 包裹。如果 LM Studio 服务在索引过程中不可用，整个 dd_documents() 调用会崩溃，导致已成功嵌入的文本块也无法写入数据库。
**建议**: 在 dd_documents() 中对 embed_documents() 调用增加异常捕获，记录失败详情并跳过失败的文档，而非让整个索引过程失败。

### N8. [P2] embed_query() 缺少重试逻辑
**文件**: core/core.py 第 77-78 行, core_qdrant.py 第 65-66 行, core_chromadb.py 第 56-57 行
`python
def embed_query(self, text: str) -> List[float]:
    return self._embed(text)
`
**问题**: embed_documents() 增加了重试机制（max_retries=2），但 embed_query() 直接调用 _embed() 没有任何重试。用户查询时的 Embedding 服务临时抖动会直接导致查询失败，而索引时同样的问题却能自动重试。
**影响**: 查询操作比索引操作更容易因临时网络问题失败。
**建议**: 为 embed_query() 也添加重试逻辑（可复用相同的 max_retries 参数）。

### N9. [P3] core_qdrant.py 缺少 _get_ocr_txt_path() 方法
**文件**: core/core_qdrant.py
**问题**: 整改将 _get_ocr_txt_path() 辅助方法添加到了 core.py 和 core_chromadb.py，但 core_qdrant.py 没有此方法，也没有 _detect_need_ocr() 或 _auto_ocr() 方法。Qdrant 版本完全没有 OCR 支持。
**影响**: 功能缺失（Qdrant 版本不支持 OCR），但由于该文件已标注为备用，影响有限。
**建议**: 同步添加 OCR 相关方法，或在文件头部废弃注释中明确说明此限制。

### N10. [P3] /api/files 接口无路径验证
**文件**: web/web_app.py 第 317-335 行
**问题**: /api/files 接口使用 os.walk(DOCUMENTS_PATH) 列出文件并返回相对路径，但没有像 /api/delete 和 /api/ocr 那样使用 secure_document_path() 验证。虽然这只是只读操作（列出文件），不涉及删除或修改，但返回的路径信息可能被攻击者用于探测服务器文件结构。
**影响**: 信息泄露风险（低），因为只返回 DOCUMENTS_PATH 内的文件。
**建议**: 风险较低，当前可接受。如需加固，可过滤掉 .rag 目录以外的系统目录。

## 五、总体评价

### 整改质量: 良好

两轮整改共修复 25 项问题（原审计 29 项中修复 20 项 + 新发现 10 项中修复 5 项），所有已声称修复项均通过代码级验证。

### 关键改进
1. **安全性**: 路径遍历漏洞已修复（含 Windows 大小写绕过）
2. **可靠性**: Embedding 服务不可用时不再污染数据库，改为重试+明确异常
3. **健壮性**: 文本分块死循环风险已消除（三个文件同步）
4. **检索质量**: 混合检索向量距离归一化已修复
5. **代码一致**: 三个核心文件的关键修复已同步

### 剩余风险
- core_qdrant.py 作为备用文件，仍有 OCR 路径 bug（N6）和缺少 OCR 支持的问题（N9）
- dd_documents() 缺少对 embed_documents() 的异常捕获（N7），服务不可用时会整体崩溃
- embed_query() 没有重试逻辑（N8），查询比索引更容易因临时故障失败

这三项均不影响日常功能（LM Studio 服务正常时不会触发），但在服务不稳定时可能导致用户体验下降，建议下一版本处理。
