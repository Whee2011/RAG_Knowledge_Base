# 🔄 RAG 混合检索功能说明

**版本：** v3.4  
**更新日期：** 2026-04-21  
**功能：** 智能混合检索（向量 + 关键词）

---

## 📖 功能概述

RAG 知识库系统现已支持**智能混合检索**功能，可根据问题复杂度自动选择最优检索策略：

| 问题类型 | 检索策略 | 说明 |
|---------|---------|------|
| **简单问题** | 向量检索 | 快速、精准，适合事实查询 |
| **复杂问题** | 混合检索 | 向量+ 关键词，提升召回率 |

---

## 🎯 智能判断规则

系统自动判断问题复杂度，规则如下：

### 判定为复杂问题（使用混合检索）

1. **问题长度 > 15 字**
   - 例："2026 年 2 月电费比 1 月增长了多少百分比？"

2. **包含复杂关键词**
   - 为什么、如何、怎么、比较、分析、关系
   - 区别、影响、原因、过程、步骤、比、对比、差异

3. **多个问题**
   - 包含 2 个以上问号（`?` 或 `？`）

### 判定为简单问题（使用向量检索）

不满足以上条件的均为简单问题：
- 例："2 月电费是多少？"
- 例："机器人型号是什么？"

## 🔧 技术实现

### 混合检索算法

采用 **RRF (Reciprocal Rank Fusion)** 倒数排名融合：

```
最终分数 = α × (1/(k+ 向量排名)) + (1-α) × (1/(k+BM25 排名))
```

**参数设置：**
- `α = 0.5`（向量 50% + 关键词 50%）
- `k = 60`（RRF 常数）

### 检索流程

```
用户问题
    ↓
智能判断（简单/复杂）
    ↓
简单问题 → 向量检索 → 返回结果
复杂问题 → 混合检索 → 向量检索 + BM25 → RRF 融合 → 返回结果
    ↓
LLM 生成答案 → 返回用户
```

## 💻 使用方式

### 方式 1：Web 界面（自动）

无需任何配置，Web 界面自动使用智能检索：

```
http://localhost:5000
```

### 方式 2：命令行

```bash
# 使用智能 query 方法（推荐）
python core\core.py query "问题内容"

# 示例
python core\core.py query "为什么 A 公司比 B 公司利润高？"
```

### 方式 3：Python API

```python
from core.core import KnowledgeBase

kb = KnowledgeBase()

# 智能查询（自动判断检索策略）
result = kb.query("问题内容", top_k=5)

# 强制使用向量检索
result = kb.query_with_sources("问题内容", top_k=5)

# 强制使用混合检索
result = kb.query_hybrid("问题内容", top_k=5, alpha=0.5)

print(f"答案：{result.answer}")
print(f"来源：{result.sources}")
print(f"耗时：{result.query_time:.2f}秒")
```

##  最佳实践

### 适用场景

**混合检索适合：**
- ✅ 复杂推理问题（"为什么...？"）
- ✅ 对比分析问题（"A 和 B 的区别？"）
- ✅ 多条件查询（"2026 年 2 月比 1 月增长多少？"）
- ✅ 关键词匹配（精确术语查询）

**向量检索适合：**
- ✅ 简单事实查询（"2 月电费是多少？"）
- ✅ 语义查询（"机器人型号？"）
- ✅ 快速响应场景

### 参数调优

如需调整混合权重，可修改 `alpha` 参数：

```python
# 向量权重更高（适合语义查询）
result = kb.query_hybrid("问题", top_k=5, alpha=0.7)

# 关键词权重更高（适合精确匹配）
result = kb.query_hybrid("问题", top_k=5, alpha=0.3)
```

## ❓ 常见问题

### Q1: 混合检索比向量检索慢多少？

**A:** 约慢 0.5-2 秒（取决于文档数量），因为需要：
1. 构建 BM25 索引（首次）
2. 执行两次检索（向量+BM25）
3. 分数融合计算

### Q2: 是否需要重新索引？

**A:** 不需要。混合检索复用现有向量索引，只需：
1. 安装 `rank-bm25` 包
2. 重启服务即可

### Q3: 如何查看使用了哪种检索策略？

**A:** 查看日志输出：
```
[INFO] 简单问题，使用向量检索
[INFO] 复杂问题，使用混合检索
```
